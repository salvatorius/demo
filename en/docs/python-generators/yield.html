Yield

La instrucción yield es la que utilizaremos para definir un generador en Python. Cuando el intérprete de python encuentra la instrucción yield dentro de una función, esa función producirá entonces un generador, que podremos utilizar para iterar sobre esta secuencia de valores de retorno o respuestas (una nueva en cada solicitud que se le haga)

Sintaxis básica​

La sintaxis básica de la instrucción yield es la siguiente:

def mi_generador():
    # instrucciones que nos permiten obtener
    # el nuevo valor en cada nueva iteración
    yield valor

La función mi_generador (o el nombre que decidamos darle) es ahora un generador que devuelve el valor especificado en la instrucción yield. Cuando se llama a la función mi_generador, lo que se devuelve no es un objeto con la respuesta directamente (en el ejemplo, no obtenemos ese valor que aparentemente retorna el yield) Como hemos dicho, se devuelve un objeto generator, que se puede utilizar para, a partir de éste, iterar sobre la secuencia de valores que esperábamos obtener de esa función.

gen = mi_generador()
# ↑ gen no es el valor que podríamos pensar que retorna la función "mi_generador"
# en realidad lo que obtenemos es un «generator»
type(gen)
-> <class 'generator'>

Ahora, cada vez que se itera sobre el objeto generador, se ejecuta la verdadera función definida por nuestro generador, recorriendo las líneas de nuestro código en forma "normal", hasta que se encuentra la instrucción yield, con la cual fijamos en qué punto se devuelve el valor especificado, y se detiene la ejecución hasta la siguiente invocación (o iteración) sobre el generador.

Para ello contamos con (entre otras) 2 formas principales de solicitar cada siguiente valor de nuestro generador:

  • Opción A: Iterar directamente sobre el generador:
for item in gen:
    print(item)

o (análogamente) usando List Comprehension:

print ([item for item in gen])
  • Opción B: Solicitar, a discreción, el siguiente elemento disponible en la secuencia, cuando se requiera, con la función next:
gen_io = mi_generador()
print ('Primer elemento', next(gen_io))
# mas código
print ('El siguiente elemento en la secuencia es: ', next(gen_io))

lo cual también podríamos hacer dentro de un bucle:

while (condicion):
    # garantizamos que en algún momento
    # sí se cumpla la condición de parada
    # y pedimos el siguiente resultado disponible:
    print('Siguiente elemento: ', next(gen_io))

Es importante destacar que la función generador no se ejecuta completamente cuando se llama, sino que se detiene en donde sea que se consiga la instrucción yield. Se pausa en ese punto (devolviendo lo que enviemos como argumento del yield) y sólo se reanuda su ejecución (la instrucción o control de flujo que hubiese seguido de no haberse topado con el yield) desde ese punto, la siguiente vez que se llama al generador.

Eso, entre otras cosas, quiere decir también que bien podríamos tener más código aún después de la instrucción yield (y éste se va a ejecutar consecuentemente una vez que se reanude la ejecución tras la siguiente iterción o solicitud del next) o se reanuda el control de flujo, si es que el yield (por ejemplo) se encontraba en medio de un bucle for.
